RamiBot Admin Facility - User Guide

Welcome to the RamiBot Admin Facility! This guide will help you manage and modify the Intent Classifier Feature efficiently.

Key Features

  • View Intents: Review existing intents and their properties, including the last modified date and the user who made the changes.
  • Add New Intents: Easily add new intents for RamiBot to learn and understand.
  • Edit Intents: Update existing intents to ensure the chatbot accurately interprets user messages.
  • Delete Intents: Remove obsolete or incorrect intents.
  • Train Model: Train RamiBot's model after making any changes to the intents.
  • Test Model: Verify RamiBot's responses by testing the model against various inputs.

How to Use

1. Viewing Intents

Navigate to the "Modify Intents" page to view a list of all current intents. You can edit or delete intents directly from this page.

2. Adding New Intents

Click the "Add New Intent" button to create a new intent. Provide the required details, such as the intent name, examples of user inputs, and corresponding responses.

3. Editing Intents

To edit an existing intent, click on the "Edit" link next to the intent you wish to modify. Update the necessary fields and save your changes.

4. Deleting Intents

Click "Delete" next to any intent to remove it. You will be asked to confirm before the intent is permanently deleted.

5. Training the Model

Once you've made changes to the intents, click "Train Model" to retrain RamiBot. Training will take around 2-3 mins so please be patient.

6. Testing the Model

After training, use the "Test Model" feature to input sample messages and see how RamiBot responds based on the updated model.

Best Practices

Ensure that the intents you create or modify are specific and cover a wide range of user inputs.
Changes will only reflect after re-training. Always test the new changes to ensure RamiBot will only output correct information.
It is recommended to only write keywords or phrases in the Patterns field when creating or editing intents.
Avoid using similar patterns or responses across multiple intents to prevent confusion in the model.
